Objectif : Transformer le WebView actuel en une expérience avancée avec panneau manipulable + adblock intégré type uBlock.

1. UI / UX WebView
- Le WebView ne doit PLUS être en plein écran.
- Il doit apparaître comme un panneau (bottom sheet) qui sort depuis le bas.
- Doit être :
  - draggable (glisser vers le haut / bas)
  - avec positions : réduit / moyen / plein écran
  - animation fluide (spring ou easeOut)
- Ajouter :
  - un handle (poignée) en haut (petite barre arrondie)
  - un bouton "X" pour fermer (coin haut droit)
- Le fond derrière doit être légèrement assombri (backdrop)

2. Header du WebView
Créer une barre en haut avec :
- bouton retour
- bouton refresh
- bouton fermer
- icône AdBlock (important)

Design :
- style moderne (dark, glassmorphism léger)
- coins arrondis
- cohérent avec une app premium

3. Système AdBlock intégré
Pas d’extensions Chrome. Implémenter un système natif :

A. Blocage réseau (priorité)
- Intercepter toutes les requêtes HTTP
- Bloquer si URL match une blocklist

Utiliser ces domaines (liste principale) :
doubleclick.net
googlesyndication.com
googleadservices.com
googletagservices.com
googletagmanager.com
ads.yahoo.com
adservice.google.com
amazon-adsystem.com
adnxs.com
taboola.com
outbrain.com
popads.net
adsterra.com
propellerads.com
revcontent.com
media.net
yandexadexchange.net
smartadserver.com
rubiconproject.com
openx.net
criteo.com
adsrvr.org

B. Blocage scripts / éléments
- Injecter JS après chargement page :
  - supprimer <iframe>, <script>, <div> contenant :
    id/class = ad, ads, banner, sponsor, popup
- Masquer overlays et popups

C. Blocage CSS
- appliquer display:none sur :
  .ad
  .ads
  .banner
  .sponsor
  .popup
  iframe[src*="ads"]

4. Menu AdBlock (clic sur icône)
Créer un panneau comme un mini menu flottant avec options :

- Activer / désactiver adblock (toggle)
- Bloquer un élément manuellement (mode sélection)
- Masquer un élément (CSS hide)
- Supprimer un élément (remove DOM)
- Voir les éléments bloqués (log)
- Réinitialiser règles

5. Mode sélection (important)
Quand activé :
- survol / clic sur élément page
- surligner en rouge
- clic = proposer :
  - bloquer domaine
  - masquer élément
  - supprimer élément

6. Stockage
- sauvegarder règles utilisateur (local storage / DB)
- recharger automatiquement à chaque session

7. Performance
- éviter lag scroll
- blocage doit être async et rapide
- ne pas casser le site principal

8. Bonus (si possible)
- compteur de pubs bloées
- whitelist par site
- animation douce pour menu

Résultat attendu :
Un WebView style navigateur intégré premium, manipulable comme TikTok/YouTube mini player + adblock avancé type uBlock intégré.
